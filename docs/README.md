# AegisFlow Documentation Hub

Welcome to the AegisFlow documentation directory. This directory contains resources to help both humans and AI Assistants understand the core principles, architecture, and execution flow of the AegisFlow Application Security Posture Management (ASPM) platform.

## Available Documents

1. **[AI Context (`context.md`)](context.md)**
   - High-level overview of AegisFlow.
   - Project goals, target audience, and core philosophy.
   - Guidelines for AI understanding and extension.

2. **[System Architecture (`architecture.md`)](architecture.md)**
   - Detailed structural layout of the project.
   - Explanation of the three pillars: Server, Pipeline, and Dashboard.
   - Data synchronization flow between components.

3. **[Execution Workflow (`workflow.md`)](workflow.md)**
   - Step-by-step breakdown of the `run_pipeline.sh` orchestrator.
   - Explanation of each scanning stage (SAST, SCA, DAST, etc.).
   - Insights into the AI Triage engine and the Policy Gate.

4. **[Security Policies & Governance (`policies.md`)](policies.md)**
   - Policy Enforcement rules (Quality Gate logic).
   - SLAs and Exception handling.
   - Role-based dashboard perspectives.

5. **[Threat Modeling (`threat-model.md`)](threat-model.md)**
   - Security of the pipeline itself (Secrets leakage, compromised runner, supply chain).
   - Secure-by-Design and App Threat Modeling mappings.

6. **[Data Schemas & Contracts (`data-schemas.md`)](data-schemas.md)**
   - Explains the JSON structures (`full_report.json`, `policy_result.json`) that act as the API contracts between components.

7. **[Setup & Execution (`setup-and-run.md`)](setup-and-run.md)**
   - Instructions for running the project locally via Docker Compose and `run_mac.sh`.

8. **[Extending AegisFlow (`extending-aegisflow.md`)](extending-aegisflow.md)**
   - A step-by-step developer guide on how to integrate new security tools (scanners) into the pipeline without breaking the UI or AI Triage.

9. **Case Study Reports**
   - `DevSecOps_CaseStudy_Report.pdf`
   - `DevSecOps_CaseStudy_Report.html`
   - These are output artifacts generated by the pipeline, provided here as examples of the final case study generation step.
